## Migrating from Intercom

<Steps>
  <Step title="Export from Intercom">
    **Export contacts:**
    1. Intercom â†’ Settings â†’ Data export
    2. Select "Contacts"
    3. Download CSV

    **Export conversations:**
    1. Settings â†’ Data export
    2. Select "Conversations"
    3. Download JSON
  </Step>

  <Step title="Prepare data">
    Map Intercom fields to Team Inbox:
    
    | Intercom | Team Inbox |
    |----------|------------|
    | user.email | contact.email |
    | user.phone | contact.phone |
    | user.name | contact.name |
    | tags | contact.tags |
    | conversations | (import as notes) |
  </Step>

  <Step title="Import to Team Inbox">
    ```bash
    # Import contacts
    npm run import:contacts intercom-contacts.csv

    # Import conversation history as notes
    npm run import:history intercom-conversations.json
    ```
  </Step>
</Steps>

## Migrating WhatsApp Integration

<Steps>
  <Step title="Get WhatsApp Business API access">
    If not already using WhatsApp Business API:
    1. Apply through Meta
    2. Get business verified
    3. Register phone number

    [WhatsApp setup â†’](/deployment/whatsapp-setup)
  </Step>

  <Step title="Update webhook URL">
    Point WhatsApp webhook to Team Inbox:
    
    **Old:** `https://intercom.com/webhook/...`
    **New:** `https://your-domain.com/api/webhooks/whatsapp`

    Change in Meta Developer Console
  </Step>

  <Step title="Test thoroughly">
    1. Send test messages
    2. Verify receipt in Team Inbox
    3. Reply and confirm delivery
    4. Check all team members can access
  </Step>

  <Step title="Switch over">
    **Gradual rollout recommended:**
    - Week 1: Use both systems in parallel
    - Week 2: Primary on Team Inbox, Intercom as backup
    - Week 3: Fully on Team Inbox
  </Step>
</Steps>

## Common Migration Questions

<AccordionGroup>
  <Accordion title="Will I lose conversation history?">
    **Conversation data:**
    - Import as notes/context
    - Maintain in old system for reference
    - Export before canceling old service

    **Going forward:**
    - All new conversations in Team Inbox
    - Full history preserved
  </Accordion>

  <Accordion title="How long does migration take?">
    **Timeline:**
    - Setup: 2-4 hours
    - Data import: 1-2 hours
    - Team training: 1-2 hours
    - Testing: 1-2 hours
    - **Total: 1 day for small team**

    **Larger teams:** Plan 1-2 weeks for full migration
  </Accordion>

  <Accordion title="Can I run both systems in parallel?">
    Yes! Recommended approach:
    - Week 1: Parallel (both active)
    - Week 2: Gradual shift (Team Inbox primary)
    - Week 3: Team Inbox only

    Allows team to adapt gradually
  </Accordion>

  <Accordion title="What about existing customer expectations?">
    **Communicate change:**
    - Send announcement before migration
    - Update website/help docs
    - Set auto-reply in old system

    **Example message:**
    ```
    We've upgraded to faster WhatsApp support!
    Please message us on WhatsApp: +1234567890
    ```
  </Accordion>
</AccordionGroup>
